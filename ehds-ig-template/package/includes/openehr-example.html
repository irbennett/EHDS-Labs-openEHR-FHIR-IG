{% comment %} openehr-example.html — render openEHR composition examples as FLAT / STRUCTURED / CANONICAL tabs. Files must be placed in input/includes/openehr-input/templates/{templateDir}/compositions/ using the naming convention: {base}.flat.json — openEHR FLAT (path-key) JSON {base}.structured.json — openEHR STRUCTURED JSON {base}.canonical.json — openEHR CANONICAL JSON The templateDir is resolved automatically from site.data.openehr_examples by matching base. It can also be passed explicitly via the template parameter (takes precedence). Markup matches ehrexplorer.css + openehr-init.js (.openehr-format-tabs, .openehr-tab-link, .openehr-tab-pane). Parameters: base (required) Base name shared by the format files, e.g. "ips-composition-example". template (optional) Override the templateDir lookup. title (optional) Human-readable heading shown above the sections. flat Set to true to render the FLAT JSON tab. structured Set to true to render the STRUCTURED JSON tab. canonical Set to true to render the CANONICAL JSON tab. The first available format (flat, then structured, then canonical) is the initially active tab. Each tab has a copy button using the publisher's bundled ClipboardJS (.btn-copy).
