{% comment %} openehr-example.html — render openEHR composition examples as collapsible sections. Files must be placed in input/includes/openehr-input/templates/{templateDir}/compositions/ using the naming convention: {base}.flat.json — openEHR FLAT (path-key) JSON {base}.structured.json — openEHR STRUCTURED JSON {base}.canonical.json — openEHR CANONICAL JSON The templateDir is resolved automatically from site.data.openehr_examples by matching base. It can also be passed explicitly via the template parameter (takes precedence). Parameters: base (required) Base name shared by the format files, e.g. "ips-composition-example". template (optional) Override the templateDir lookup. title (optional) Human-readable heading shown above the sections. flat Set to true to render the FLAT JSON section. The .flat.json file must exist. structured Set to true to render the STRUCTURED JSON section. The .structured.json file must exist. canonical Set to true to render the CANONICAL JSON section. The .canonical.json file must exist. The first available format is expanded by default; others are collapsed.
