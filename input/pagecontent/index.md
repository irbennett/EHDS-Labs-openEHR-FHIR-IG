### Introduction

This implementation guide is designed to support **data exchange using [openEHR](https://www.openehr.org/)**. The FHIR artifacts defined here serve as the foundation for interoperability within the project, alongside clinical information models expressed in openEHR.

### Specification

This guide publishes profiles, value sets, and examples for laboratory and related clinical data. Use these definitions when mapping between openEHR archetypes/templates and FHIR resources, or when building APIs that need to align with both ecosystems.

For background on openEHR, see the [openEHR specifications](https://specifications.openehr.org/) and community resources. For FHIR, see the [HL7 FHIR](https://www.hl7.org/fhir/) standard.
