# openEHR Composition Examples

openEHR composition instances showing how clinical data is structured according to openEHR archetypes and templates.
Each example can be POSTed to an openEHR Clinical Data Repository to persist the composition.

Each example is available in one or more serialisation formats:

- **FLAT JSON** — path-key format (used by most CDR REST APIs for POST/PUT)
- **STRUCTURED JSON** — nested format following the Reference Model hierarchy
- **CANONICAL JSON** — full canonical representation including all RM metadata
